# MagicQC Mail Platform

Self-hosted enterprise email platform for MagicQC employees.

## Architecture

| Component | Technology | Port |
|-----------|-----------|------|
| Mail Server | [Stalwart](https://stalw.art/) (SMTP + IMAP + JMAP + Spam Filter) | 25, 465, 587, 993, 4190 |
| Webmail | Next.js 16 + React 19 + Tailwind CSS 4 | 3000 (container) → 13000 (host) |
| Reverse Proxy | Caddy v2 (host-level, auto-HTTPS) | 80, 443 |
| Protocol | JMAP (RFC 8620/8621) | via Stalwart HTTP |

```
Internet
  │
  ├── :25/:465/:587/:993 ──→ Stalwart (Docker)
  │
  └── :443 (HTTPS)
        │
        Caddy (host)
        ├── mail.magicqc.com   → localhost:13000 (Next.js webmail)
        ├── jmap.magicqc.com   → localhost:18080 (Stalwart JMAP)
        ├── admin.magicqc.com  → localhost:18080 (Stalwart admin)
        ├── magicqc.com        → localhost:8000  (Laravel app)
        └── robionix.com       → localhost:8080  (WordPress)
```

## Quick Start

### Prerequisites

- VPS: Ubuntu 24.04, Docker + Docker Compose v2
- DNS: magicqc.com records pointing to `121.52.149.158`

### 1. First-time Server Setup

```bash
ssh -p 222 nutechadmin@121.52.149.158
git clone <repo-url> ~/Deploy/mail-platform
cd ~/Deploy/mail-platform
./setup-server.sh   # Installs Caddy, opens firewall ports
```

### 2. Configure DNS

```bash
./setup-dns.sh   # Prints all required DNS records for Namecheap
```

Add the printed records at [Namecheap DNS Management](https://ap.www.namecheap.com/).

### 3. Configure Environment

```bash
cp .env.example .env
nano .env
```

Required variables:
- `MAIL_HOSTNAME=mail.magicqc.com`
- `MAIL_DOMAIN=magicqc.com`
- `NEXT_PUBLIC_JMAP_URL=https://jmap.magicqc.com`
- `STALWART_ADMIN_PASSWORD=<strong-password>`

### 4. Deploy

```bash
./deploy.sh
```

### 5. Post-Deploy: Create First Admin

Open `https://admin.magicqc.com` and log in with:
- Username: `admin`
- Password: value from `STALWART_ADMIN_PASSWORD`

Then create employee accounts (e.g. `abdullah@magicqc.com`).

## Development

```bash
cd frontend
npm install
npm run dev        # http://localhost:3000
```

The JMAP client points to `NEXT_PUBLIC_JMAP_URL`. For local development against a running Stalwart instance:

```bash
NEXT_PUBLIC_JMAP_URL=http://localhost:18080 npm run dev
```

## Email Security

| Layer | Implementation |
|-------|---------------|
| SPF | `v=spf1 a mx ip4:121.52.149.158 -all` |
| DKIM | Auto-generated by Stalwart |
| DMARC | `p=reject` with reporting |
| MTA-STS | Enforce mode via Caddy endpoint |
| TLS | Automatic via Caddy (Let's Encrypt) |
| Spam Filter | Stalwart built-in (Bayesian + blocklists + SPF/DKIM/DMARC checks) |
